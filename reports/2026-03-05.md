## Archived Papers - Deep Dive

### Memex(RL): Scaling Long-Horizon LLM Agents via Indexed Experience Memory

- **Paper ID**: 2603.04257v1
- **URL**: https://arxiv.org/abs/2603.04257v1

该论文旨在解决大型语言模型（LLM）代理在处理长时间任务时因有限上下文窗口而面临的瓶颈问题。随着任务轨迹增长，工具输出和中间推理难以在上下文中保留，导致上下文过长并超出预算，同时远距离证据难以有效利用。现有方法通常通过截断或总结来缩短上下文，但这些方法不可避免地丢失信息，压缩或丢弃了关键证据。

为此，作者提出了一种名为Memex的索引化经验记忆机制，通过维护简洁的结构化摘要和稳定索引来构建紧凑的工作上下文，同时将完整的交互数据存储在外部经验数据库中。代理可以根据需要反向索引以恢复精确的过去证据，从而在不丢失信息的情况下压缩上下文。进一步，作者开发了MemexRL强化学习框架，优化了写入和读取行为，使代理能够学习如何总结、归档、索引以及检索数据。

实验结果表明，在复杂的长时间任务中，使用MemexRL训练的Memex代理显著提高了任务成功率，同时显著减少了工作上下文的长度。此外，理论分析证明了Memex循环在保持决策质量的同时，通过有限反向索引实现了有效的上下文计算。

与现有方法相比，该研究通过引入索引化记忆机制避免信息丢失，提供了一种更可靠的长时间记忆解决方案。这项工作不仅在提升LLM代理的长时间任务能力方面具有重要意义，还为未来研究指明了方向，包括进一步优化记忆机制和扩展至更广泛的应用场景。

### Mozi: Governed Autonomy for Drug Discovery LLM Agents

- **Paper ID**: 2603.03655v1
- **URL**: https://arxiv.org/abs/2603.03655v1

《Mozi: Governed Autonomy for Drug Discovery LLM Agents》提出了一种名为Mozi的双层架构，用于解决药物发现领域中工具增强型大语言模型（LLM）代理面临的两大核心挑战：工具使用的无约束治理和长时间任务中的可靠性不足。当前药物研发流程复杂且依赖性强，传统的LLM代理容易因早期阶段的幻觉（hallucination）导致后续结果不可复现，从而引发系统性失败。

Mozi的核心方法包括两层架构：控制层（Control Plane）和工作流层（Workflow Plane）。控制层通过监督-工作者的层级体系对工具使用进行角色隔离，限制操作在约束的动作空间内，并引入基于反思的重新规划机制。工作流层将药物发现的关键阶段（如靶点识别、先导化合物优化）模块化为可组合的状态图，并通过严格的数据契约和人类参与（HITL）检查点，确保在高不确定性决策点上的科学有效性。该架构基于“自由推理用于安全任务，结构化执行用于长时间任务”的设计原则，提供了内置的鲁棒性机制和可追溯性，以彻底消除错误积累。

实验结果表明，Mozi在PharmaBench基准测试中显著优于现有方法，展现出卓越的任务协调能力。此外，通过端到端的治疗案例研究，Mozi成功探索了广阔的化学空间，严格执行毒性筛选，并生成了具有竞争力的候选化合物，验证了其在药物发现中的实际应用潜力。

与现有方法相比，Mozi的创新在于将生成式AI的灵活性与计算生物学的确定性相结合，通过多层次治理和模块化设计实现了科学推理和可靠执行的平衡。未来，该方法有望进一步扩展至其他高风险科学领域，为自动化科研提供更高的可信度与可控性。

### Beyond Task Completion: Revealing Corrupt Success in LLM Agents through Procedure-Aware Evaluation

- **Paper ID**: 2603.03116v1
- **URL**: https://arxiv.org/abs/2603.03116v1

该论文旨在解决当前基于大型语言模型（LLM）的智能体评估方法中存在的局限性，即过于关注任务完成情况，而忽略了任务完成的过程是否符合规范。作者提出了一个新的评估框架——程序感知评估（Procedure-Aware Evaluation, PAE），通过结构化观察和一致性关系分析，全面评估智能体在任务执行中的效用性、效率、交互质量以及程序完整性，并通过多维度门控机制识别和排除“腐败成功”（即表面完成任务但存在程序违规的结果）。

在方法创新方面，PAE通过定义智能体的观察、沟通和执行之间的关系，建立了一个多轴评估体系，能够捕捉任务完成过程中的非冗余失败模式，例如效用掩盖了可靠性问题，速度不代表精确性，简洁性无法保证意图的遵循。此外，PAE还通过门控机制对腐败成功进行分类否定，从而显著改变模型的排名和通过率。

实验结果表明，当前主流LLM智能体在tau-bench基准测试中的通过率存在显著腐败成功现象，27%-78%的成功案例存在交互或程序完整性违规。具体而言，GPT-5在策略、执行和意图维度上均有分布式错误；Kimi-K2-Thinking的违规集中于策略忠实性和合规性；Mistral-Large-3主要表现为忠实性失败。此外，研究还揭示了基准设计中的结构性缺陷，包括任务范围不足、奖励信号矛盾以及模拟器生成的意外成功。

与现有方法相比，该工作不仅关注任务完成结果，还深入探讨了任务完成的质量和规范性，提出了更全面的评估标准。未来，该框架有潜力在高风险场景中广泛应用，例如医疗诊断或自动驾驶领域，同时为改进基准设计和智能体开发提供了重要方向。
